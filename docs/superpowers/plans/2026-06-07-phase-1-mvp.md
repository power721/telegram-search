# Phase 1 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the tg-provider Phase 1 MVP: local config/runtime setup, SQLite+FTS5 persistence, Gin API, Telegram login/session/channel/history sync through gotd, and searchable message history.

**Architecture:** Use small internal Go packages with repository/service/API boundaries. Keep Telegram behind an interface so the database, search, handlers, and sync logic are tested without real Telegram network access.

**Tech Stack:** Go 1.26, Gin, SQLite FTS5 via `github.com/mattn/go-sqlite3`, gotd/td, zap, lumberjack, YAML v3.

---

## File Structure

- `go.mod`: module and dependency declarations.
- `cmd/tg-provider/main.go`: startup, signal handling, shutdown.
- `internal/config/config.go`: config loading, defaults, validation, runtime paths.
- `internal/config/config_test.go`: config behavior tests.
- `internal/logger/logger.go`: zap loggers with lumberjack rotation.
- `internal/db/db.go`: SQLite open/close, PRAGMAs, migration runner, transaction helper.
- `internal/db/migrations.go`: Phase 1 schema, FTS5 table, triggers, indexes.
- `internal/db/db_test.go`: migration and FTS tests.
- `internal/model/*.go`: model structs and status constants.
- `internal/repository/*.go`: account/channel/message/link/status repositories.
- `internal/repository/repository_test.go`: repository integration tests.
- `internal/link/extractor.go`: generic URL extractor for Phase 1.
- `internal/link/extractor_test.go`: extraction tests.
- `internal/session/session.go`: deterministic session paths.
- `internal/telegram/client.go`: Telegram interfaces and DTOs.
- `internal/telegram/gotd_client.go`: gotd-backed login/channel/history implementation.
- `internal/channel/service.go`: channel sync service.
- `internal/history/service.go`: history sync service.
- `internal/history/service_test.go`: transactional history sync tests with fake Telegram client.
- `internal/search/service.go`: search/latest/links query service.
- `internal/search/service_test.go`: search behavior tests.
- `internal/api/router.go`: Gin route registration and handler dependencies.
- `internal/api/handlers.go`: status, login, account, channel, search, latest, links handlers.
- `internal/api/handlers_test.go`: API handler tests.
- `README.md`: quickstart and Phase 1 limitations.
